# CPX 시나리오 운영 및 향후 확장 명세서

## 1. 개요

본 문서는 초기 MVP에 탑재할 발열 케이스 운영 방식과 향후 케이스 확장 방식을 정의한다. 모든 발열 케이스는 동일한 **발열 채점표 Yes/No 100점 기준**으로 채점한다.


## 채점 기준 요약

- 기준: 발열 채점표 예시
- 방식: 40개 항목 전체 Yes/No
- 총점: `Yes 개수 / 40 × 100`
- 가중치: 없음
- 부분점수: 없음
- 기존 `잘함/보통/미흡` 표기는 사용하지 않음


## 2. 초기 탑재 시나리오 — 발열 8케이스

1. **급성 백혈병**: 발열, 출혈 경향, 빈혈 증상 확인.
2. **약물열**: 최근 약물 시작, 상대적 서맥, 감염 focus 부재 확인.
3. **전신홍반루푸스**: 미열, 관절통, 나비모양 발진, 광과민성 확인.
4. **상기도 감염**: 상부/하부 호흡기 증상을 분리하여 확인.
5. **급성 신우신염**: 배뇨 증상, 옆구리 통증, CVA 압통 확인.
6. **감염성 심내막염**: 치과 시술력, 심질환 병력, 지속 미열, 심잡음 확인.
7. **연조직염**: 국소 발적, 열감, 부종, 압통, 외상력 확인.
8. **말라리아**: 주기적 발열, 여행/군 복무/야외활동/모기 물림 확인.

## 3. 케이스 운영 원칙

### 3.1 공통 문진 구조

모든 발열 케이스는 아래 순서의 문진을 유도한다.

1. 발열 시작 시기와 지속 기간
2. 발열 양상과 체온 측정 여부
3. 주변 유사 증상자
4. 전신 증상
5. 계통별 동반 증상
6. 상부/하부 호흡기 증상 분리
7. 탈수 증상
8. 위험 요인
9. 여행력, 접촉력, 음식 복용
10. 야외활동, 벌레 물림
11. 과거력, 약물력, 사회력, 가족력, 직업/환경

### 3.2 공통 진찰 구조

모든 발열 케이스는 활력징후, 결막/공막, 구강/인후, 경부 림프절, 흉부, 복부, 피부, 관절, CVA 압통, 수막자극징후를 Yes/No로 평가할 수 있어야 한다.

### 3.3 공통 교육 구조

환자교육은 추정진단, 법정감염병 보고 가능성, 진단 계획, 치료 계획, 응급 상황 및 감염병 예방 교육으로 구성한다.

## 4. 확장성 아키텍처

신규 케이스를 생성할 때도 채점 기준은 바꾸지 않는다. 케이스별 환자 데이터만 바뀌고, 평가 엔진은 동일한 40개 Yes/No 항목으로 작동한다.

1. AI가 신규 케이스 초안을 생성한다.
2. 임상팀이 양성/음성 소견을 검수한다.
3. 개발팀이 환자 페르소나와 진찰 반환값을 JSON으로 등록한다.
4. 평가 엔진은 공통 40개 항목으로 자동 채점한다.
