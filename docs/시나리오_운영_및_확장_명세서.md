# CPX 시나리오 운영 및 향후 확장(Scale-up) 명세서

## 1. 개요
본 문서는 초기 MVP(Minimum Viable Product) 모델에 탑재될 **대표적인 '발열(Fever)' 8가지 케이스**를 정의하고, 향후 다른 진료과 및 증상의 시나리오를 AI 기반으로 자동 생성하여 확장하는 아키텍처를 명세합니다.

## 2. 초기 탑재 시나리오 (발열 8케이스)
제공된 전문 CPX 자료를 바탕으로, 의대생이 반드시 감별해야 하는 발열의 핵심 원인 질환 8가지를 1차 시나리오로 구축합니다.

1.  **Case 1. 급성 백혈병 (Acute Leukemia)**
    *   **학습 목표**: 발열과 동반된 출혈 경향(점상출혈), 빈혈 증상을 청취하고 혈액 질환을 의심할 수 있는가.
2.  **Case 2. 약물열 (Drug Fever)**
    *   **학습 목표**: 최근 추가된 투약력(통풍약 등)을 확인하고, 고열임에도 양호한 전신 상태(상대적 서맥)를 감별할 수 있는가.
3.  **Case 3. 전신홍반루푸스 (SLE)**
    *   **학습 목표**: 발열과 동반된 관절통, 피부 발진(나비모양 홍반) 및 광과민성 등의 자가면역 질환 특징을 파악할 수 있는가.
4.  **Case 4. 상기도 감염 (URI)**
    *   **학습 목표**: 가장 흔한 발열 원인인 호흡기 증상을 꼼꼼히 문진하고, 다른 중증 질환과 배제 진단할 수 있는가.
5.  **Case 5. 급성 신우신염 (APN)**
    *   **학습 목표**: 비뇨기계 증상(배뇨통, 빈뇨)과 측복부 통증(CVA tenderness)을 확인하여 요로 감염을 의심할 수 있는가.
6.  **Case 6. 감염성 심내막염 (Infective Endocarditis)**
    *   **학습 목표**: 과거 치과 치료력이나 심질환(판막 등) 병력을 확인하고, 지속적인 원인 미상의 발열 원인을 찾을 수 있는가.
7.  **Case 7. 연조직염 (Cellulitis)**
    *   **학습 목표**: 국소적인 피부의 부종, 열감, 통증, 홍반 등을 확인하고 외상력이나 기저질환(당뇨 등)을 확인할 수 있는가.
8.  **Case 8. 말라리아 (Malaria)**
    *   **학습 목표**: 위험 지역(군 복무, 여행력) 방문력과 오한(Chill)을 동반한 주기적인 발열 패턴을 청취할 수 있는가.

## 3. 확장성(Scale-up) 아키텍처: AI 주도 자동 케이스 생성
위 8개의 발열 케이스가 시스템 내에 완벽하게 JSON 형태로 구조화되면, 이를 바탕으로 무한한 시나리오 확장이 가능해집니다.

### 3.1. RAG 및 Few-Shot 프롬프팅
*   **원리**: 새로운 케이스(예: '기침' 5케이스) 생성을 요청하면, AI는 기존 발열 8케이스의 데이터 구조(주소, 과거력, 가족력, 환자 페르소나, 채점 루브릭 템플릿)를 RAG(검색 증강 생성)로 불러와 참고(Few-Shot)합니다.
*   **효과**: 사람이 일일이 기획하지 않아도, 기존 포맷과 동일한 품질의 완벽한 텍스트 데이터 셋을 AI가 10초 안에 생성합니다.

### 3.2. 확장 워크플로우 (Human-in-the-loop)
1.  **AI 초안 생성**: 관리자가 "급성 맹장염 시나리오를 만들어줘"라고 입력.
2.  **데이터 세팅**: AI가 병력, 신체진찰 소견, 의대생이 꼭 해야 할 질문(루브릭)을 기존 포맷에 맞춰 JSON으로 생성.
3.  **전문가 검수 (Human Review)**: 의과대학 교수진 또는 시스템 관리자가 의학적 오류 유무만 빠르게 검수 후 [승인].
4.  **DB 반영 및 배포**: 승인 즉시 플랫폼의 `Scenarios` 테이블에 등록되어 학생들이 실습 가능.

## 4. 기대 효과
*   초기 8케이스 구축 비용만 투자하면, 이후 시나리오 확장 비용은 사실상 AI API 호출 비용(건당 몇 십 원 수준)으로 수렴합니다.
*   타 진료과(소화기, 호흡기, 정신과 등) 콘텐츠 확장이 극적으로 빨라져, 플랫폼의 스케일업(Scale-up) 속도가 기존 교육 솔루션을 압도합니다.
