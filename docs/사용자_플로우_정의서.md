# 사용자 플로우 정의서

## 1. 개요

본 문서는 의대생 사용자가 플랫폼에 접속해 CPX 발열 실습을 수행하고, Yes/No 100점 채점 결과를 확인하기까지의 화면 이동 흐름을 정의한다.


## 채점 기준 요약

- 기준: 발열 채점표 예시
- 방식: 40개 항목 전체 Yes/No
- 총점: `Yes 개수 / 40 × 100`
- 가중치: 없음
- 부분점수: 없음
- 기존 `잘함/보통/미흡` 표기는 사용하지 않음


## 2. 주요 플로우

### Step 1. 진입 및 로그인

1. 랜딩 페이지에서 서비스 소개를 확인한다.
2. 이메일/비밀번호로 로그인 또는 회원가입을 진행한다.
3. 로그인 성공 시 홈 대시보드로 이동한다.

### Step 2. 시나리오 탐색 및 준비

1. 시나리오 목록에서 `발열` 케이스를 선택한다.
2. 시나리오 상세 화면에서 환자 기본 정보, 주소, 제한 시간을 확인한다.
3. 진행 모드를 선택한다.
   - **학습 모드**: 학생의 질문 흐름에 따라 AI 튜터 힌트를 제공할 수 있다.
   - **시험 모드**: AI 환자는 묻지 않은 정보를 절대 먼저 말하지 않는다.
4. 장치 체크 모달에서 마이크와 스피커를 확인한다.

### Step 3. 실시간 CPX 실습 진행

1. AI 환자가 주소만 먼저 말한다.
2. 학생은 음성 또는 텍스트로 문진한다.
3. AI 환자는 질문받은 내용에만 답한다.
4. 신체진찰 요청이 들어오면 해당 케이스의 소견을 텍스트로 반환한다.
5. 제한 시간 종료 또는 진료 종료 버튼 클릭 시 세션을 종료한다.

### Step 4. 결과 분석 및 피드백 확인

결과 화면은 아래 구조로 표시한다.

#### 4.1 총점 카드

```text
총점 75.0 / 100
Yes 30개 / No 10개
채점 기준: 발열 채점표 예시 기반 Yes/No 40항목
```

#### 4.2 Yes/No 체크리스트

- 40개 항목 전체를 표로 보여준다.
- 각 행은 `평가 항목`, `Yes/No`, `근거 발화`, `피드백`으로 구성한다.
- 기존 `잘함/보통/미흡` UI는 사용하지 않는다.

#### 4.3 누락 항목 피드백

- `No` 항목을 우선순위로 묶어서 보여준다.
- 예: “여행력, 접촉력, 음식 복용을 확인하지 않아 감염성 발열 감별이 부족했습니다.”
- 각 피드백은 “왜 필요한지”를 함께 설명한다.

#### 4.4 대화 기록

- 전체 Transcript를 타임스탬프와 함께 제공한다.
- 각 Yes 항목에 연결된 근거 문장을 하이라이트한다.

## 3. 예외 플로우

- 네트워크 끊김: 대화 일시중지 및 재연결 안내.
- 음성 인식 실패: “마이크에 소리가 입력되지 않고 있습니다. 다시 말씀해 주세요.” 안내.
- 중도 이탈: “진행 중인 진료를 취소하시겠습니까? 기록이 저장되지 않습니다.” 모달 표시.
