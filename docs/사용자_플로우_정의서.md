# 사용자 플로우 정의서 (User Flow)

## 1. 개요
본 문서는 의대생(사용자)이 플랫폼에 접속하여 CPX 실습을 수행하고 피드백을 확인하기까지의 주요 화면 이동 흐름과 사용자 경험을 정의합니다. 프론트엔드 개발 시 네비게이션 및 컴포넌트 설계의 기준이 됩니다.

## 2. 주요 플로우 (Main Flow)

### Step 1. 진입 및 로그인
1.  **[랜딩 페이지]**: 서비스 소개 및 로그인/회원가입 유도.
2.  **[로그인/회원가입 화면]**: 이메일 및 비밀번호 입력.
    *   *오류 시*: "이메일 또는 비밀번호가 일치하지 않습니다." 알림.
3.  **[홈 화면 (대시보드)]**: 로그인 성공 시 이동.
    *   최근 연습 기록 요약, 추천 시나리오 노출.

### Step 2. 시나리오 탐색 및 준비
1.  **[시나리오 목록 화면]**: '시나리오 탐색' 메뉴 클릭 시 진입.
    *   과목, 증상(예: 발열) 기반 필터링 제공.
2.  **[시나리오 상세(브리핑) 화면]**: 특정 시나리오 클릭 시 진입.
    *   환자 정보, 주소(Chief Complaint), 제한 시간 확인.
    *   **[실습 모드 선택]**:
        *   ① **실습 모드**: 실제 국가고시와 동일한 환경으로 진행하며, 대화 중 AI 튜터의 실시간 가이드(Tutor)가 함께 제공됨
        *   ② **능동적 문진 강제 설계 모드**: 묻기 전엔 절대 답하지 않는 엄격한 환자
    *   `[연습 시작]` 버튼 노출.
3.  **[장치 체크 모달]**: 시작 버튼 클릭 시 마이크 권한 요청 및 볼륨 레벨 테스트.

### Step 3. 실시간 CPX 실습 진행
1.  **[대화 세션 화면]**:
    *   **초기 상태**: 환자의 첫 마디 음성 재생 및 텍스트 표시.
    *   **사용자 발화**: 마이크를 켜고 질문 진행.
    *   **AI 응답**: STT -> LLM -> TTS. 환자 페르소나에 맞춰 응답.
    *   **[실습 모드 한정] 실시간 튜터링**: "현재 상기도 감염이 의심되나, 말라리아 배제를 위해 여행력을 확인해 보세요" 등 실시간 사고과정 안내.
2.  **[세션 종료]**: 제한 시간 종료 또는 `[진료 종료]` 버튼 클릭 시 종료.

### Step 4. 결과 분석 및 피드백 확인
1.  **[로딩 화면]**: AI가 대화 기록(Transcript)과 PMR 기반 루브릭을 교차 검증하여 채점.
2.  **[세션 결과 화면] (Explainable Feedback)**:
    *   **종합 점수**: 병력청취, 의사소통, 환자교육 점수 표시.
    *   **임상 사고과정 시각화**: 학생의 질문 흐름을 시각적으로 도식화 (예: 발열 -> 호흡기 확인 -> 여행력 미확인).
    *   **이유가 있는 피드백**: "여행력을 묻지 않아 감점됨"이 아니라 "말라리아 배제를 위해 여행력 확인이 필수였습니다" 형태의 튜터링 피드백 제공.
    *   **대화 전체 기록(Transcript)**: 타임스탬프 및 발화 내용 확인.
3.  **[종료 및 이동]**: `[대시보드로 돌아가기]` 또는 `[다른 시나리오 연습하기]` 버튼 클릭.

## 3. 예외 플로우 (Exception Flow)
*   **세션 중 네트워크 끊김**: "네트워크 연결이 불안정합니다. 대화가 일시 중지되었습니다." 팝업 노출 및 재연결 시도.
*   **음성 인식 실패(무음 5초 지속)**: "마이크에 소리가 입력되지 않고 있습니다. 다시 말씀해 주시겠어요?" 안내 메시지 표시.
*   **중도 이탈**: 세션 중 뒤로가기 클릭 시 "진행 중인 진료를 취소하시겠습니까? 기록이 저장되지 않습니다." 경고 모달 노출.
