본 문서는 프로젝트 전체 기획 흐름을 한눈에 파악하기 위한 마스터 인덱스입니다. 프론트엔드, 백엔드, AI 프롬프트, 해커톤 발표 문서는 모두 아래의 공통 채점 기준을 따른다.

## 공통 채점 기준 v2 — 발열 채점표 예시 기반 Yes/No 100점

본 프로젝트의 자동 채점은 **발열 채점표 예시**를 기준으로 통일한다. 기존의 `잘함/보통/미흡`, 항목별 가중치, PMR 다축 가중 평가는 제거하고, 모든 항목을 **Yes / No**로만 판단한다.

### 점수 산식

```text
총 항목 수 = 40개
Yes = 1, No = 0
최종 점수 = (Yes 개수 / 40) × 100
항목당 점수 = 2.5점
```

- 별도 가중치는 두지 않는다.
- 부분점수는 두지 않는다.
- 학생이 같은 의미의 질문·진찰·설명을 명확히 수행하면 `Yes`로 인정한다.
- 환자가 먼저 말한 내용을 학생이 능동적으로 확인하지 않은 경우는 원칙적으로 `No`로 처리한다.
- 최종 피드백은 점수보다 `No` 항목을 중심으로 “무엇을 놓쳤는지”와 “왜 필요한지”를 설명한다.
- 채점 응답 최적화를 위해 LLM은 Yes 항목 번호와 짧은 근거만 생성하고, 서버가 전체 40개 결과와 No 항목을 정규화한다.

## 1. 프로젝트 핵심 비전

- **목표**: 비용과 시공간 제약 없이 의대생이 CPX 발열 케이스를 반복 실습할 수 있는 AI 표준화환자 플랫폼 제공.
- **핵심 차별점**: AI가 먼저 정보를 누설하지 않는 표준화환자 가드레일과, 발열 채점표 기반의 명확한 Yes/No 자동 채점.
- **피드백 방향**: 단순 점수 제공이 아니라 `No` 항목을 기준으로 누락 질문, 누락 진찰, 누락 교육을 설명한다.

## 3. 전체 서비스 플로우 요약

1. 사용자가 로그인 후 발열 시나리오를 선택한다.
2. AI 환자는 주소만 먼저 말하고, 학생이 묻지 않은 과거력·가족력·사회력·동반증상은 말하지 않는다.
3. 학생은 문진 중 신체진찰 모달을 열어 체위, 손소독, 도구, 진찰 항목을 직접 선택한다.
4. 일반 진찰은 5초, 환자 협조 진찰은 10초, 체위 변경은 2초가 실제 세션 타이머에서 즉시 차감된다.
5. 수행한 신체진찰 소견은 대화 기록과 AI 환자 맥락에 반영된다.
6. 세션 종료 후 Transcript와 신체진찰 로그를 40개 항목과 대조한다.
7. 각 항목은 Yes/No로만 판단한다.
8. 최종 점수는 `Yes 개수 / 40 × 100`으로 계산한다.
9. 결과 화면은 총점, Yes/No 체크리스트, 누락 항목별 이유 기반 피드백을 제공한다.
