# AI CPX 채점 평가 루브릭 및 피드백 명세서 — Yes/No 100점 버전

## 1. 개요

본 문서는 CPX 발열 케이스의 최종 자동 채점 기준을 정의한다. 기존의 PMR 기반 다축 평가, 항목별 가중치, `잘함/보통/미흡` 등급은 구현 기준에서 제외하고, **발열 채점표 예시를 그대로 Yes/No 체크리스트로 변환한 100점 방식**을 적용한다.


---

## 공통 채점 기준 v2 — 발열 채점표 예시 기반 Yes/No 100점

본 프로젝트의 자동 채점은 **발열 채점표 예시**를 기준으로 통일한다. 기존의 `잘함/보통/미흡`, 항목별 가중치, PMR 다축 가중 평가는 제거하고, 모든 항목을 **Yes / No**로만 판단한다.

### 점수 산식

```text
총 항목 수 = 40개
Yes = 1, No = 0
최종 점수 = (Yes 개수 / 40) × 100
항목당 점수 = 2.5점
```

- 별도 가중치는 두지 않는다.
- 부분점수는 두지 않는다.
- 학생이 같은 의미의 질문·진찰·설명을 명확히 수행하면 `Yes`로 인정한다.
- 환자가 먼저 말한 내용을 학생이 능동적으로 확인하지 않은 경우는 원칙적으로 `No`로 처리한다.
- 최종 피드백은 점수보다 `No` 항목을 중심으로 “무엇을 놓쳤는지”와 “왜 필요한지”를 설명한다.

## 2. 자동 채점 원칙

### 2.1 인정 기준

- 학생 발화가 평가 항목과 의미상 동일하면 표현이 달라도 `Yes`로 인정한다.
- 예: “최근 여행 다녀오셨어요?”, “요즘 어디 다녀온 곳 있으세요?”는 모두 여행력 확인으로 인정한다.
- 진찰은 학생이 명확히 시행 의사를 밝히거나 수행했다고 기록된 경우 `Yes`로 인정한다.
- 교육은 환자가 이해할 수 있는 수준으로 설명해야 `Yes`로 인정한다.

### 2.2 불인정 기준

- 학생이 직접 묻지 않았는데 환자가 말한 정보는 학생의 능동 문진으로 보지 않는다.
- 질문이 지나치게 모호해 항목 판단이 어려우면 `No`로 처리한다.
- “다 괜찮으세요?”처럼 여러 항목을 한 번에 뭉뚱그린 질문은 구체성이 없으면 해당 세부 항목을 인정하지 않는다.
- 부분 수행, 불완전 수행, 추정 수행은 별도 부분점수 없이 `No`로 처리한다.

## 4. 피드백 문장 생성 규칙

피드백은 `No` 항목을 중심으로 생성한다.

| 누락 항목 유형 | 피드백 문장 예시 |
|---|---|
| 여행력 미확인 | “발열 환자에서는 말라리아, 장티푸스 등 감염성 원인을 배제하기 위해 최근 여행력을 확인해야 합니다.” |
| 탈수 증상 미확인 | “발열이 지속되면 탈수나 순환 저하가 동반될 수 있으므로 목마름, 어지러움, 소변량 감소를 확인해야 합니다.” |
| CVA 압통 미확인 | “발열과 옆구리 통증이 있으면 신우신염 가능성을 확인하기 위해 늑골척추각 압통을 확인해야 합니다.” |
| 환자교육 미흡 | “추정 진단뿐 아니라 검사 계획, 치료 계획, 응급 상황 시 재내원 기준까지 설명해야 합니다.” |
| 공감 부족 | “환자의 불안이나 통증을 한 번 이상 언급하며 공감하면 PPI 점수를 확보할 수 있습니다.” |

## 5. 개발 구현 포인트

### 5.1 Transcript 기반 판정

1. 전체 대화 기록을 문장 단위로 분리한다.
2. 각 문장을 40개 평가 항목과 의미 매칭한다.
3. 항목별 근거 문장을 저장한다.
4. 근거가 있으면 `Yes`, 없으면 `No`로 기록한다.
5. `Yes 개수 / 40 × 100`으로 최종 점수를 계산한다.

### 5.2 저장 필드 예시

```sql
score_total numeric,
yes_count integer,
total_items integer default 40,
scoring_mode text default 'fever_checklist_yes_no_100_no_weight',
score_items jsonb,
missed_items jsonb,
feedback_summary text
```
