# 백엔드 개발 태스크 칸반 (Task Breakdown - Hybrid Architecture)

본 문서는 **Supabase(BaaS) + FastAPI(Microservice)** 하이브리드 아키텍처를 기반으로 백엔드 개발을 위해 쪼갠 3가지 핵심 에픽(Epic)과 세부 태스크입니다.

---

## Epic 1. Supabase 인프라 구축 (DB, Auth, Storage)
> **Goal**: 벡엔드 코딩 없이 프론트엔드와 직접 통신할 수 있는 완벽한 DB 및 인증 인프라를 구축합니다.

*   **[Task 1-1] Supabase 프로젝트 생성 및 SQL 배포**: `schema.sql`을 실행하여 `Organizations`, `Users`, `Scenarios`, `Sessions`, `Rubrics` 등의 테이블 생성.
*   **[Task 1-2] RLS (Row Level Security) 정책 설정**: 학생은 본인의 `Sessions`만 볼 수 있고, 어드민은 모든 데이터를 볼 수 있도록 DB 단에서 보안 정책 설정.
*   **[Task 1-3] Supabase Auth 연동**: 이메일 로그인/회원가입 기능 활성화 및 `public.Users` 테이블과 `auth.users` 간의 Trigger 연동.
*   **[Task 1-4] Storage 버킷 생성**: 학생들의 진료 음성 파일을 저장할 `audio_records` 퍼블릭 버킷 생성.

## Epic 2. FastAPI 실시간 AI 통신 엔진 구축 (WebSocket)
> **Goal**: 딜레이가 없는 실시간 양방향 오디오 스트리밍 및 AI 환자 페르소나 제어.

*   **[Task 2-1] FastAPI 프로젝트 초기화**: `uvicorn`, `fastapi`, `supabase-py` 라이브러리 세팅 및 로컬 서버 구동.
*   **[Task 2-2] LLM 프롬프트 빌더(Builder) 구현**: Supabase DB에서 시나리오 데이터를 읽어와 시스템 프롬프트(가드레일 포함)를 동적으로 생성.
*   **[Task 2-3] WebSocket 라우터 구현**: `ws://api/v1/sessions/{session_id}/stream` 엔드포인트를 열고 프론트엔드와 오디오 청크를 송수신.
*   **[Task 2-4] STT/LLM/TTS 파이프라인**: 들어온 바이너리를 STT로 텍스트 변환 -> LLM으로 환자 대답 생성 -> TTS로 음성 변환 후 프론트엔드로 즉시 전송하는 비동기 루프 완성.
*   **[Task 2-5] Supabase 비동기 로깅**: 대화가 오고 갈 때마다 FastAPI가 백그라운드에서 Supabase `Transcripts` 테이블에 기록.

## Epic 3. 평가 엔진 및 Explainable Feedback (FastAPI)
> **Goal**: 세션 종료 시나리오를 바탕으로 루브릭 채점 및 튜터링 피드백 자동 생성.

*   **[Task 3-1] 루브릭 교차 검증 알고리즘**: 세션 종료 호출 시 `Transcripts` 내용과 `Rubrics` 체크리스트를 LLM으로 교차 분석하여 점수 산출.
*   **[Task 3-2] Explainable Feedback 생성**: 학생의 질문 누락에 대해 '왜 물어봐야 했는지' 설명하는 튜터링 문구 생성.
*   **[Task 3-3] Clinical Reasoning 시각화 로직**: 학생의 질문 순서 플로우를 트리(Tree) 구조로 포맷팅.
*   **[Task 3-4] 결과 저장**: 산출된 피드백 결과 JSON을 Supabase `Feedback_Results` 테이블에 `UPDATE`.
