# 백엔드 개발 태스크 칸반 (Task Breakdown)

본 문서는 `API_명세서.md` 및 `데이터베이스_스키마_명세서.md`를 바탕으로 백엔드 기능 개발을 위해 쪼갠 5가지 에픽(Epic)과 세부 태스크입니다.
이 문서를 바탕으로 GitHub Issues 또는 Jira 티켓을 생성하여 개발을 진행합니다.

---

## Epic 1. 데이터베이스 및 인프라 기본 셋업
> **Goal**: PostgreSQL 등 RDBMS 구축 및 ORM 세팅, 공통 예외 처리기 구현.

*   **[Task 1-1] DB 마이그레이션 스크립트 작성**: `Users`, `Organizations`, `Subscriptions` 테이블 생성.
*   **[Task 1-2] 핵심 도메인 테이블 생성**: `Scenarios`, `Rubrics`, `Sessions`, `Transcripts`, `Feedback_Results` 엔티티 매핑.
*   **[Task 1-3] AWS S3 버킷 설정 및 연동**: 음성 파일 저장을 위한 Storage 서비스 구축.
*   **[Task 1-4] 글로벌 예외 처리(Global Exception Handler)**: API 공통 에러 응답 규격(400, 401, 403, 404, 500) 구현.

## Epic 2. 인증(Auth) 및 B2B 구독 시스템 구현
> **Goal**: JWT 기반 인증 로직 및 기관 소속 여부에 따른 권한 제어.

*   **[Task 2-1] Auth API 구현**: `POST /auth/register`, `POST /auth/login`, `GET /auth/me` 엔드포인트 작업. (JWT 발급 및 검증 인터셉터 적용)
*   **[Task 2-2] B2B 조직 매핑**: 회원가입 시 `organization_code`를 받아 `Organizations` 테이블과 FK 연결 로직 구현.
*   **[Task 2-3] 구독 검증 로직**: 세션 시작 시 `Subscriptions` 테이블을 조회하여 플랜 유효성 검사. (만료 시 403 에러 리턴)

## Epic 3. AI 파이프라인 연동 (프롬프트 & STT/TTS)
> **Goal**: 3가지 모드 제어 및 외부 AI API(OpenAI 등) 연결 클래스 구축.

*   **[Task 3-1] LLM 프롬프트 빌더(Builder) 구현**: `Scenarios` 테이블의 데이터를 읽어와 시스템 프롬프트(가드레일 포함)를 동적으로 생성.
*   **[Task 3-2] 3가지 실습 모드 제어**: Request의 `mode` (LEARNING/ACTIVE/EXAM) 값에 따라 AI Tutor 힌트 포함 여부 분기 처리.
*   **[Task 3-3] STT/TTS 모듈 인터페이스 연동**: 음성을 텍스트로(STT), 텍스트를 음성으로(TTS) 변환하는 외부 서비스(Whisper, ElevenLabs 등) API 통신 클라이언트 구현.

## Epic 4. 실시간 음성 스트리밍 서버 구축 (WebSocket)
> **Goal**: 딜레이 최소화를 위한 양방향 소켓 통신 구현 및 대화 기록(Transcript) 저장.

*   **[Task 4-1] WebSocket 핸들러 설정**: `ws://api/v1/sessions/{session_id}/stream` 엔드포인트 라우팅 및 세션 연결/해제 관리 로직.
*   **[Task 4-2] 오디오 청크 파이프라인 처리**: 프론트에서 넘어오는 바이너리 음성을 모아 STT로 넘기고, LLM 응답을 받아 TTS로 변환 후 다시 바이너리로 쏘는 스트림 루프 구현.
*   **[Task 4-3] 비동기 대화 로깅 & 오디오 저장**: 대화가 발생할 때마다 `Transcripts` 테이블에 기록하고, 환자와 학생의 음성을 S3에 비동기로 업로드하여 `audio_url` 업데이트.

## Epic 5. 평가 엔진 알고리즘 및 관리자(Admin) API
> **Goal**: 룰 기반/LLM 기반 채점 알고리즘 및 CRUD 어드민 기능.

*   **[Task 5-1] 루브릭 교차 검증 로직**: 세션 종료 후 `Transcripts` 전체 내용과 `Rubrics`의 체크리스트를 비교하여 점수(병력청취/의사소통 등) 산출.
*   **[Task 5-2] Explainable Feedback 생성**: 학생의 대화 흐름을 분석하여 '왜 물어봐야 했는지' 설명하는 튜터링 문구 자동 생성.
*   **[Task 5-3] Clinical Reasoning 시각화 데이터 매핑**: 학생의 질문 순서를 트리(Tree) 또는 그래프 노드 형태로 포맷팅하여 프론트엔드로 전달.
*   **[Task 5-4] Admin API (조회/등록)**: `Organizations`, `Scenarios`, `Users` 통계 데이터 및 등록 기능 백엔드 개발.
