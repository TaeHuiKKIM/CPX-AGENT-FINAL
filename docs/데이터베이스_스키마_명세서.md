# 데이터베이스 스키마 명세서 (ERD Draft)

## 1. 개요
본 문서는 AI 표준화환자 CPX 플랫폼의 관계형 데이터베이스 구조를 정의합니다. 사용자, 시나리오, 루브릭, 학습 세션(연습 기록)을 중심으로 설계되었습니다.

## 2. 테이블 명세

### 2.1 Users (사용자)
의대생 및 관리자의 계정 정보를 저장합니다.
*   `user_id` (PK, UUID): 사용자 고유 ID
*   `email` (String, Unique): 로그인 이메일
*   `password_hash` (String): 암호화된 비밀번호
*   `name` (String): 사용자 이름
*   `university` (String): 소속 대학교 (선택)
*   `grade` (Integer): 학년
*   `role` (Enum): `STUDENT` | `ADMIN`
*   `created_at` (Timestamp): 가입일

### 2.2 Scenarios (CPX 시나리오)
진료 연습의 주제가 되는 환자 시나리오 데이터입니다.
*   `scenario_id` (PK, UUID): 시나리오 고유 ID
*   `title` (String): 시나리오 제목 (예: "급성 복통을 호소하는 50대 남성")
*   `department` (String): 진료 과목 (예: "소화기내과")
*   `difficulty` (Enum): `EASY` | `MEDIUM` | `HARD`
*   `patient_info` (JSONB): 환자 상세 정보 (이름, 나이, 과거력, 가족력 등) - 프롬프트에 주입됨
*   `is_active` (Boolean): 활성화 여부
*   `created_by` (FK, Users.user_id): 생성한 관리자 ID

### 2.3 Rubrics (채점 루브릭)
시나리오별 채점 기준을 저장합니다.
*   `rubric_id` (PK, UUID): 루브릭 고유 ID
*   `scenario_id` (FK, Scenarios.scenario_id): 연결된 시나리오 ID
*   `version` (Integer): 루브릭 버전 관리
*   `criteria` (JSONB): 항목별 채점 기준 (병력청취, 의사소통, 환자교육 별 세부 배점 및 체크리스트)
*   `created_at` (Timestamp): 생성일

### 2.4 Sessions (연습 세션)
사용자가 연습한 한 회차의 정보를 기록합니다.
*   `session_id` (PK, UUID): 세션 고유 ID
*   `user_id` (FK, Users.user_id): 연습한 사용자 ID
*   `scenario_id` (FK, Scenarios.scenario_id): 선택한 시나리오 ID
*   `start_time` (Timestamp): 시작 시간
*   `end_time` (Timestamp): 종료 시간
*   `status` (Enum): `COMPLETED` | `INCOMPLETE`

### 2.5 Transcripts (대화 전사 기록)
세션 중 발생한 실시간 대화 기록(STT/TTS 텍스트)을 저장합니다.
*   `transcript_id` (PK, UUID)
*   `session_id` (FK, Sessions.session_id)
*   `speaker` (Enum): `USER` (의대생) | `AI` (표준화환자)
*   `content` (Text): 발화 내용
*   `timestamp` (Timestamp): 발화 발생 시간

### 2.6 Feedback_Results (피드백 및 평가 결과)
세션 종료 후 AI가 루브릭을 바탕으로 채점한 결과를 저장합니다.
*   `result_id` (PK, UUID)
*   `session_id` (FK, Sessions.session_id)
*   `rubric_id` (FK, Rubrics.rubric_id): 채점에 사용된 루브릭 버전
*   `score_history_taking` (Float): 병력청취 점수
*   `score_communication` (Float): 의사소통 점수
*   `score_education` (Float): 환자교육 점수
*   `total_score` (Float): 총점
*   `strengths` (JSONB): 강점 하이라이트 내역
*   `weaknesses` (JSONB): 약점 및 개선 코칭 내역
*   `created_at` (Timestamp): 피드백 생성 시간
