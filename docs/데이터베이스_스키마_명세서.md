# 데이터베이스 스키마 명세서 (ERD Draft - Enterprise Ver.)

## 1. 개요
본 문서는 AI 표준화환자 CPX 플랫폼의 관계형 데이터베이스 구조를 정의합니다. 사용자, 시나리오, 루브릭, 학습 세션을 비롯하여 **B2B SaaS(기관/학교 구독)** 및 **실시간 음성 데이터 스토리지** 관리를 포함하는 엔터프라이즈급 아키텍처로 설계되었습니다.

## 2. 테이블 명세

### 2.1 Organizations (기관 및 학교) [NEW]
B2B 고객사 관리를 위한 테이블입니다.
*   `organization_id` (PK, UUID): 기관 고유 ID
*   `name` (String): 기관명 (예: 서울대학교 의과대학)
*   `domain` (String): 기관 이메일 도메인 (예: snu.ac.kr)
*   `created_at` (Timestamp): 등록일

### 2.2 Subscriptions (구독 및 결제) [NEW]
기관 또는 개인 사용자의 서비스 구독 상태를 관리합니다.
*   `subscription_id` (PK, UUID)
*   `organization_id` (FK, Organizations.organization_id, Nullable)
*   `user_id` (FK, Users.user_id, Nullable)
*   `plan_type` (Enum): `BASIC` | `PREMIUM` | `ENTERPRISE`
*   `status` (Enum): `ACTIVE` | `EXPIRED` | `CANCELED`
*   `start_date` (Date)
*   `end_date` (Date)

### 2.3 Users (사용자)
의대생 및 관리자의 계정 정보를 저장합니다.
*   `user_id` (PK, UUID): 사용자 고유 ID
*   `organization_id` (FK, Organizations.organization_id): 소속 기관 ID (선택)
*   `email` (String, Unique): 로그인 이메일
*   `password_hash` (String): 암호화된 비밀번호
*   `name` (String): 사용자 이름
*   `grade` (Integer): 학년
*   `role` (Enum): `STUDENT` | `PROFESSOR` | `ADMIN`
*   `created_at` (Timestamp): 가입일

### 2.4 Scenarios (CPX 시나리오)
진료 연습의 주제가 되는 환자 시나리오 데이터입니다.
*   `scenario_id` (PK, UUID): 시나리오 고유 ID
*   `title` (String): 시나리오 제목 (예: "급성 복통을 호소하는 50대 남성")
*   `department` (String): 진료 과목 (예: "소화기내과")
*   `difficulty` (Enum): `EASY` | `MEDIUM` | `HARD`
*   `patient_info` (JSONB): 환자 상세 정보 (이름, 나이, 과거력, 가족력 등) - 프롬프트에 주입됨
*   `is_active` (Boolean): 활성화 여부
*   `created_by` (FK, Users.user_id): 생성한 관리자 ID

### 2.5 Rubrics (채점 루브릭)
시나리오별 채점 기준을 저장합니다.
*   `rubric_id` (PK, UUID): 루브릭 고유 ID
*   `scenario_id` (FK, Scenarios.scenario_id): 연결된 시나리오 ID
*   `version` (Integer): 루브릭 버전 관리
*   `criteria` (JSONB): 항목별 채점 기준 (병력청취, 의사소통, 환자교육 별 세부 배점 및 체크리스트)
*   `updated_by` (FK, Users.user_id): 마지막 수정자 ID
*   `created_at` (Timestamp): 생성일

### 2.6 Sessions (연습 세션)
사용자가 연습한 한 회차의 정보를 기록합니다.
*   `session_id` (PK, UUID): 세션 고유 ID
*   `user_id` (FK, Users.user_id): 연습한 사용자 ID
*   `scenario_id` (FK, Scenarios.scenario_id): 선택한 시나리오 ID
*   `mode` (Enum): `LEARNING` | `ACTIVE` | `EXAM` (실습 모드)
*   `start_time` (Timestamp): 시작 시간
*   `end_time` (Timestamp): 종료 시간
*   `status` (Enum): `COMPLETED` | `INCOMPLETE` | `ERROR`

### 2.7 Transcripts (대화 전사 및 오디오 기록)
세션 중 발생한 실시간 대화 텍스트 및 음성 파일 경로를 저장합니다.
*   `transcript_id` (PK, UUID)
*   `session_id` (FK, Sessions.session_id)
*   `speaker` (Enum): `USER` (의대생) | `AI` (표준화환자)
*   `content` (Text): 발화 내용 텍스트 (STT/TTS 결과)
*   `audio_url` (String): 발화 음성 파일 저장 경로 (S3/GCS URL) [NEW]
*   `timestamp` (Timestamp): 발화 발생 시간

### 2.8 Feedback_Results (피드백 및 평가 결과)
세션 종료 후 AI가 루브릭을 바탕으로 채점한 결과를 저장합니다.
*   `result_id` (PK, UUID)
*   `session_id` (FK, Sessions.session_id)
*   `rubric_id` (FK, Rubrics.rubric_id): 채점에 사용된 루브릭 버전
*   `score_history_taking` (Float): 병력청취 점수
*   `score_communication` (Float): 의사소통 점수
*   `score_education` (Float): 환자교육 점수
*   `total_score` (Float): 총점
*   `strengths` (JSONB): 강점 하이라이트 내역
*   `weaknesses` (JSONB): 약점 및 개선 코칭 내역
*   `clinical_reasoning_flow` (JSONB): 임상 사고과정 시각화를 위한 플로우 데이터
*   `explainable_feedback` (JSONB): 단순 감점이 아닌 '왜 질문해야 했는지' 설명하는 튜터링 피드백 내역
*   `created_at` (Timestamp): 피드백 생성 시간
