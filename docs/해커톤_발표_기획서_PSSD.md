# 2026 의료 AI·SW 융합 해커톤 발표 기획서 — 최종 데모 반영판

## 발표 핵심 메시지

> “저희 서비스는 AI 표준화환자와 문진하고, 신체진찰을 실제 CPX 절차처럼 수행한 뒤, 발열 채점표 40개 항목을 Yes/No로 자동 채점해 누락된 문진·진찰·교육을 이유와 함께 알려주는 CPX 학습 플랫폼입니다.”


## 채점 기준 요약

- 기준: 발열 채점표 예시
- 방식: 40개 항목 전체 Yes/No
- 총점: `Yes 개수 / 40 × 100`
- 가중치: 없음
- 부분점수: 없음
- 모델은 Yes 항목 번호와 짧은 근거만 반환하고, 서버가 40개 체크리스트를 복원해 채점 시간을 줄임


## 1. Problem

의대생은 실제 표준화환자와 충분히 반복 연습하기 어렵다. 특히 CPX는 “무엇을 물었는가”뿐 아니라 “묻지 않은 정보는 얻을 수 없는가”, “신체진찰을 올바른 순서와 도구로 수행했는가”, “제한 시간 안에 필요한 절차를 끝냈는가”가 함께 평가된다. 기존 챗봇형 연습은 대화는 가능해도 신체진찰 플로우와 시간 손실까지 반영하기 어렵다.

## 2. Solution

### 2.1 AI 표준화환자

AI 환자는 주소만 먼저 말하고, 학생이 묻기 전까지 과거력, 가족력, 사회력, 약물력, 동반증상을 누설하지 않는다.

### 2.2 플로우 기반 신체진찰 모듈

- 학생은 모달에서 체위, 손소독, 도구, 진찰 항목을 직접 선택한다.
- 청진·타진·촉진·수기 진찰은 즉시 시간 차감이 반영된다.
- 체위 변경은 2초 로딩과 함께 실제 세션 타이머에서 차감된다.
- 폐 청진, 복부 청진, CVAT 등 수행 항목별 소견이 대화 기록으로 전달되어 AI 환자와 최종 채점에 반영된다.

### 2.3 발열 채점표 기반 자동 채점

- 발열 채점표 예시의 40개 항목을 그대로 사용한다.
- 모든 항목은 Yes/No로만 채점한다.
- 최종 점수는 `Yes 개수 / 40 × 100`으로 계산한다.
- 항목별 가중치와 부분점수는 없다.
- 채점 속도를 위해 LLM은 Yes 항목 번호와 근거만 간결하게 반환하고, 서버가 No 항목과 전체 체크리스트를 정규화한다.

### 2.4 이유가 있는 피드백

단순히 “여행력 미확인”이라고만 보여주지 않고, “말라리아, 장티푸스 등 감염성 발열 감별을 위해 필요했다”처럼 이유를 함께 설명한다.

## 3. Scale-up

발열 8케이스의 환자 데이터는 케이스마다 다르지만, 채점 엔진은 동일한 40개 Yes/No 기준을 사용한다. 이후 기침, 복통, 흉통 등 다른 주제로 확장할 때도 먼저 해당 주제의 채점표를 구조화한 뒤 동일한 방식으로 자동 채점할 수 있다.

## 4. Development

- 텍스트 입력 및 브라우저 STT/TTS 기반 AI 표준화환자 대화
- AI 환자 정보공개 가드레일
- WebSocket 기반 FastAPI-Gemini 응답 처리
- 신체진찰 모달: 체위 변경, 손소독, 기물 선택, 청진/촉진/타진/수기 수행
- 수행 즉시 세션 타이머 차감: 일반 진찰 5초, 환자 협조 진찰 10초, 체위 변경 2초
- Transcript 및 신체진찰 로그 기반 40항목 Yes/No 자동 채점
- Yes 항목만 LLM이 반환하고 서버가 전체 결과를 복원하는 빠른 채점 구조
- 결과 리포트: 총점, 영역별 레이더 차트, 강점/보완점, 상세 피드백, 체크리스트

## 5. 발표 데모 시나리오

1. 로그인 후 발열 케이스를 선택한다.
2. 시험 모드로 시작하면 AI 환자는 주소 외 정보를 먼저 누설하지 않는다.
3. 학생이 문진 질문을 입력하면 AI 환자가 짧고 자연스럽게 응답한다.
4. 신체진찰 버튼을 열어 청진기를 선택하고 폐 청진을 수행한다.
5. 수행 즉시 타이머가 5초 줄고, 소견이 타임라인과 대화 기록에 남는다.
6. 체위를 변경하면 “체위 변경 중입니다” UI가 2초간 표시되고 시간이 즉시 차감된다.
7. 진료 종료 후 Yes/No 40항목 채점 결과와 누락 이유를 확인한다.

## 6. 한 줄 비교표

| 구분 | 기존 서비스 | 우리 서비스 |
|---|---|---|
| 환자 응답 | 정보 누설 가능 | 묻기 전 정보 제공 금지 |
| 신체진찰 | 텍스트 입력 또는 생략 | 체위·도구·손소독·순서·시간을 플로우로 기록 |
| 채점 방식 | 키워드 중심 | 발열 채점표 40항목 Yes/No + 신체진찰 로그 |
| 점수 산식 | 불명확하거나 가중치 기반 | Yes 개수 / 40 × 100 |

| 채점 속도 | 긴 전체 JSON 생성 | Yes 항목만 생성 후 서버 정규화 |
| 확장 | 케이스 수 중심 | 채점표 구조화 기반 확장 |
